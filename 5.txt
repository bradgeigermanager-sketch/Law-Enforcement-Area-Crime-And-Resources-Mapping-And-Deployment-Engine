## 3. System Architecture Breakdown

* Multi-Channel Status Notifications (Tab 1): Replaces slow manual notification chains with programmatic endpoints. It formats raw model outputs into structured JSON dictionaries (for Slack markdown rendering) or clean text string blocks (for Twilio character limits), pushing updates out to field supervisors automatically whenever an area updates to Tier 3.
* Web-Pollable RSS/XML Data Engine (Tab 2): Modern web apps often use data-heavy polling pipelines that can overwhelm networks in low-bandwidth areas. This module bundles active system exceptions into an optimized, plain-text <rss> syntax schema.
* Idle Vehicle Display Feeds: Vehicle terminals can query this unformatted string block via simple GET requests. This allows older hardware or idle field screens to decode and scroll incoming threat alerts continuously without needing full UI rendering pipelines.
