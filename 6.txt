def calculate_haversine_distance(lat1, lon1, lat2, lon2):
    R = 6371000 # Earth radius in meters
    p1, p2 = math.radians(lat1), math.radians(lat2)
    dp = math.radians(lat2 - lat1)
    dl = math.radians(lon2 - lon1)
    a = math.sin(dp/2)**2 + math.cos(p1)*math.cos(p2) * math.sin(dl/2)**2
    return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1 - a))

### 3. Structural Operational Architecture

*   **Cascading Threat Auto-Escalation Engine (`Tab 1`)**: Instead of analyzing risk in isolation, the platform uses a nested conditional rule engine to combine model forecasts with infrastructure states. If a sector's expected volume climbs (\(\ge 7.0\)) at the same time network degradation takes out multiple surrounding sensors, the area automatically escalates to `TIER_3_CRITICAL`, which triggers emergency rules in the dispatcher.
*   **Multi-Origin Variable Routing (`Tab 2`)**: Moving to a multi-base setup changes the optimization problem from a single array into an interconnected network matrix (\(B_{bases} \times G_{grids}\)). The Simplex engine optimizes the entire system simultaneously, preventing any single base from exceeding its fleet capacity:
    \[\sum_{g=1}^{G} \text{Units}_{b,g} \le \text{Fleet Capacity}_b\]
*   **Dynamic Tier-Based Staffing Constraints**: When the escalation engine flags a cell as `TIER_3_CRITICAL`, the routing optimizer updates its boundary parameters to force a minimum deployment tier (`bounds.append((3, 12))`). This ensures the system automatically routes units to unmonitored, high-risk sectors while managing base travel times across the network.
