# Calculate real-world geodesic path distance via the Haversine equationdef calculate_haversine_distance(lat1, lon1, lat2, lon2):
    R = 6371000 # Radius of earth in meters
    phi1, phi2 = math.radians(lat1), math.radians(lat2)
    dphi = math.radians(lat2 - lat1)
    dlambda = math.radians(lon2 - lon1)
    a = math.sin(dphi/2)**2 + math.cos(phi1)*math.cos(phi2) * math.sin(dlambda/2)**2
    return R * 2 * math.atan2(math.sqrt(a), math.sqrt(1 - a))

### 3. Core Engine Mechanics

*   **Stochastic Degradation Loop (`Tab 1`)**: Instead of assuming all hardware works perfectly, this engine uses a probability threshold comparison loop (`np.random.uniform(0, 100) < failure_probability`) to simulate packet loss and signal drops. If active sensors fall offline around a high-risk area, the system instantly catches the disruption and updates the asset's status to `CRITICAL DEGRADED BLIND SPOT`.
*   **Haversine Network Impedance Matrix (`Tab 2`)**: The platform uses the Haversine equation to calculate the true spherical distance between your central command base and target grid coordinates. It multiplies this distance by your average speed and a custom 1.4x traffic routing multiplier to estimate realistic drive times:
    \[\text{Transit Minutes} = \left( \frac{\text{Distance KM}}{\text{Speed KM/H}} \times 60 \right) \times 1.4\]
*   **Time-Bounded Matrix Boundaries**: If an area's calculated response time exceeds your `max_transit_cutoff`, the linear programming engine dynamically locks that sector's variable boundary to zero (`bounds.append((0, 0))`). This keeps resources focused within realistic operational ranges rather than chasing far-off target volumes.
