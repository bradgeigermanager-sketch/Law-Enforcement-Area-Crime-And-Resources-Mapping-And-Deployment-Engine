## 3. Core Failover Principles

* Information/Communications Scope (Tab 1): Rather than failing entirely during connection drops, the platform adjusts its messaging variables. When the central API link drops (OFFLINE), the system updates the payload parameters to switch from cloud servers to local mesh networks (SECONDARY_PEER_MESH_RADIO) and local disk storage arrays (LOCAL_ENCRYPTED_SQLITE_CACHE).
* Activity/Personnel Scope (Tab 2): Field operations can encounter bottlenecks if everyone relies on a single specialized team. To fix this, the engine checks unit capacities before dispatching resources. If the primary team's load crosses its max capability cap (tactical_capacity_counter >= 15), the engine follows your fallback rules to automatically route lower-priority events to secondary teams (Municipal Enforcement).
